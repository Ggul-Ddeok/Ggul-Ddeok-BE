# PROJECT.md

## 서비스 개요

**대상:** 마이스터고 / 특성화고 / 고졸 청년  
**목적:** 흩어진 청년 정책 정보를 한 곳에서 탐색  
**핵심:** AI 기반 "고졸 적합 정책" 필터링 및 요약 제공

---

## 서비스 플로우

```
메인 페이지
 → 정책 찾기 (모달) → 지역 선택
   → 정책 조회 요청
     → 결과 리스트 → 상세 보기
       → 저장하기 클릭
         → 로그인 유도 (카카오) → 북마크 저장
```

---

## 핵심 기능

### 정책 조회
- **데이터 소스:** 온통청년 Open API
- **처리 흐름:**
    1. Redis 캐시 조회 (`key: policies:{region}`)
    2. 캐시 miss → Open API 호출
    3. 서버 필터링 (지역 보정)
    4. AI 데이터 결합 (요약, 태그, 고졸 여부)
    5. Redis 캐싱 (TTL 24시간)
    6. 응답 반환

### 정책 리스트 화면
- 정책 카드: 정책명 / 한 줄 요약 (AI) / 태그 (취업, 교육 등)
- 전체 요약: AI가 지역 정책 트렌드 요약 제공

### 정책 상세 페이지
- 정책명, 상세 설명, 지원 대상, 신청 방법, 공식 링크
- 저장하기 (북마크) 기능

### 북마크
- 미로그인 → 카카오 로그인 유도
- 로그인 완료 → 북마크 저장
- 저장 데이터: `user_id`, `policy_id`, `policy_name`, `policy_url`

### 로그인
- 카카오 OAuth / Access Token 기반 / Refresh Token 미사용

---

## AI 기능

| 기능 | 설명 |
|------|------|
| 정책 요약 | 각 정책을 한 줄로 요약 |
| 고졸 대상 분류 | 고졸/특성화고 대상 여부 판단 (yes/no) |
| 태그 생성 | 취업, 교육, 창업, 금융 등 |
| 전체 리스트 요약 | 정책 트렌드 요약 |

---

## 필터링

- **지역 필터:** 사용자 입력 기반 / API + 서버 2단 필터링
- **고졸 대상 필터:** AI + 키워드 기반

---

## 데이터 저장 전략

### DB 저장 대상
- `User`
- `Bookmark`
- `PolicySummary` (AI 캐싱용, 선택)

### PolicySummary 스키마
```
policy_id
summary
is_highschool_target
tags
```

---

## 캐싱 전략

- **방식:** Lazy Loading (최초 요청 시 API 호출 → Redis 캐싱, 이후 Redis 사용)
- **Key:** `policies:{region}`
- **TTL:** 24시간
- **Pre-warming (선택):** 서울, 경기 등 인기 지역 미리 캐싱

---

## 예외 처리

- API 실패 시 → 기존 캐시 데이터 반환
- 데이터 없음 → "해당 지역 정책이 없습니다" 메시지

---

## 성능 전략

- API 호출 최소화
- AI 호출 최소화 (PolicySummary DB 캐싱)
- Redis 적극 활용

---

## 추후 확장 기능

- 맞춤 추천 (AI 기반)
- 관심사 필터
- 정책 마감 알림
